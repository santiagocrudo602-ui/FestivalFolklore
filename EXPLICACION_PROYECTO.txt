======================================================
     FESTIVAL DE FOLKLORE - EXPLICACIÓN DEL SISTEMA
======================================================
Este documento sirve como guía técnica para entender cómo
se construyó el prototipo del sistema, ideal para 
defender el proyecto ante docentes o evaluadores.

1. TECNOLOGÍAS UTILIZADAS
------------------------------------------------------
* Node.js: Es el entorno de ejecución del servidor. Permite ejecutar JavaScript del lado del servidor.
* Express.js: Framework de Node.js que usamos para levantar el servidor HTTP y crear todas las rutas de nuestra API (ej: /api/noches, /api/login).
* MySQL2: Es el paquete/driver que permite que nuestro servidor de Node.js se conecte y envíe consultas (SELECT, INSERT) a la Base de Datos MySQL.
* HTML5 / CSS3 / Bootstrap 5: Utilizados para diseñar el Frontend. Bootstrap nos permitió armar grillas responsivas y modales rápidamente.
* JavaScript "Vanilla": Se utilizó en el Frontend (public/js/main.js) para interactuar con el DOM, guardar datos en el navegador (localStorage) y hacer peticiones asíncronas al servidor mediante la API nativa "Fetch".

2. ARQUITECTURA DEL PROYECTO (Patrón MVC)
------------------------------------------------------
El proyecto está estructurado bajo un paradigma similar a Modelo-Vista-Controlador (MVC), lo que asegura un código limpio y separado por responsabilidades:

* /views (Vistas): Contiene los archivos HTML. Son estáticos, pero cobran vida a través de JS. Aquí la regla es "Solo Lectura", los organizadores no administran datos desde aquí para proteger la integridad del prototipo.
* /models (Modelos): Se encargan estrictamente de hablar con la Base de Datos. Cada modelo (clienteModel, entradaModel, festivalModel) contiene funciones asíncronas que ejecutan sentencias SQL puras.
* /controllers (Controladores): Es el cerebro del Backend. Reciben las peticiones (HTTP Requests) de la web, extraen los datos, llaman al Modelo correspondiente, y finalmente responden a la web con un JSON (ej: { success: true }).
* /config: Contiene la conexión a la Base de Datos.
* /public: Contiene recursos estáticos (CSS, JS, imágenes).
* server.js: Es el punto de entrada de la aplicación. Levanta el servidor en el puerto 3000 y define las rutas principales.

3. FLUJO DE FUNCIONAMIENTO CORE
------------------------------------------------------
a. Gestión de Datos: Los organizadores cargan Noches, Grupos y Sectores directamente en la Base de Datos (a través de MySQL Workbench/SQL).
b. Lectura Dinámica: Cuando un cliente abre la web, JavaScript hace un GET a "/api/noches". El Controlador busca las noches en la BD y se las devuelve a la web para dibujarlas en pantalla.
c. Registro y Login: Se piden exactamente los datos estipulados en la Base de Datos original (V1). El usuario se guarda en el 'localStorage' del navegador para simular una sesión persistente.
d. Compra de Entradas:
   - El usuario elige Tipo de Público, Sector y Cantidad.
   - Selecciona Butacas visualmente (con validación de límites).
   - Se hace un POST a "/api/entradas/comprar".
   - El Backend genera un Código de Barras alfanumérico aleatorio y seguro (Ej: FEST-2026-X8A).
   - El Backend inserta la compra en la tabla ENTRADA vinculando la butaca, el cliente y el precio.
   - El Frontend simula derivar al usuario a una pasarela de pagos externa (MercadoPago/Stripe) ya que el enunciado dicta que un tercero se encarga de la facturación.
